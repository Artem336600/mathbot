# Security verification follow-up: lint cleanup and log-leak regression guard

**Date:** 2026-03-04 22:25
**Files:** tests/webapp/test_admin_api.py, webapp/routers/attachments.py, tests/webapp/test_auth.py
**Severity:** medium

## Problem

Post-implementation verification found remaining quality gaps:
- Ruff lint errors (unused imports/variables)
- Missing regression check for secret leakage in authentication logs

This blocked full verification closure despite passing functional tests.

## Root Cause

The security hardening implementation focused on behavior and integration tests, but left:
- small static-quality leftovers after iterative edits
- no explicit test asserting that sensitive request fragments are never logged during auth validation failures

## Solution

Applied focused fixes:
- removed unused import in `tests/webapp/test_admin_api.py`
- removed unused import and variables in `webapp/routers/attachments.py`
- added `[FIX]` warning log when storage delete fails in attachment deletion flow
- added regression test `test_validate_init_data_does_not_log_sensitive_payload` in `tests/webapp/test_auth.py`
  - captures log output via Loguru sink
  - verifies reject reason is logged
  - verifies sensitive token/payload fragments are not logged

Validated with:
- `ruff check ...` -> clean
- targeted security test suite -> 30 passed

## Prevention

- Run lint (`ruff`) as a required gate after each security-hardening step.
- Add log-safety assertions for all auth/error paths that process external headers.
- During verification tasks, include a checklist item: "no sensitive value in logs" with automated assertion.
