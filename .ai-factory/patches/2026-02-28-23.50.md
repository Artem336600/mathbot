# Fix topic theory formatting and multiple test photos formatting with legacy images, duplicate XP profile fixes.

**Date:** 2026-02-28 23:50
**Files:** `bot/keyboards/mistakes.py`, `bot/utils.py`, `bot/handlers/sprint.py`, `bot/handlers/training.py`, `bot/handlers/topics.py`
**Severity:** medium

## Problem

1. The "by topic" button in the "My mistakes" menu was unwanted.
2. In the profile/training sections, re-solving the same tasks kept awarding duplicate XP (unlike the main task directory which correctly blocked duplicate XP).
3. Theory mode didn't attach its text as an image/document caption; instead, it sent the text and the document/photo separately.
4. Custom attached photos in admin interface weren't loading in sprint/training tests, as they only referenced the legacy `image_url` property instead of `AttachmentRepository`.

## Root Cause

1. The generic keyboard populated topic categories if they had mistakes.
2. The logic for preventing duplicate XP existed in `bot/handlers/topics.py`, but it wasn't copy-pasted over to `sprint_answer` and `training_answer` which triggered raw `stats_service.award_xp(uid, XP_CORRECT)` blindly.
3. Telegram enforces 1024-char limits on media captions. By default, `topics.py` sent the theory text in `safe_edit_text` before it attempted sending attachments.
4. `topics.py` successfully hooked into `AttachmentRepository` handling, but Sprint/Training modules were created prior to the update for `AttachmentRepository`, meaning their legacy codebase relied strictly on the `question.image_url` property.

## Solution

1. Removed the entire logic for dynamically rendering the category topics inside `mistakes_menu_keyboard`.
2. Incorporated `await ProgressRepository.get_solved_ids` ahead of time to verify `already_solved` and award correct points/no points for re-solving in sprint/training modes.
3. Created `bot/utils.py` exported function `get_question_media`, centralising attachment handling. We integrated this helper across both training and sprint pipelines to retrieve attached admin photos. 
4. Rewrote theory doc fetching flow! Now, if the text is <1000 chars, it sends `InputMediaPhoto` with `caption=text` rather than two separate Telegram messages (as long as a photo/attachment gets pulled from repo).

## Prevention

- Ensure duplicated business logic (like handling attached media files and duplicate score preventing mechanisms) operates over globally maintained utility/repo files so feature changes cascade cleanly across game modes.
- Remember to accommodate Telegram API constraints (e.g. caption max length = 1024 chars), as text content combined with a photo gives users higher QoE than fragmented content.
