Формулировка проблемы

Рынок онлайн-образования перенасыщен сложными курсами, но не предлагает инструментов для быстрого, "бесшовного" решения задач внутри привычных каналов коммуникации (мессенджеров). Это создает высокий порог входа для пользователей, которым нужна моментальная консультация, а не полноценное обучение

  

Почему сейчас

- **Тренд на микрообучение:** Люди привыкли потреблять контент короткими сессиями.

- **Экономия внимания:** Пользователь не хочет покидать Telegram, чтобы решить одну задачу.

  

# Юзер сторис

  

> [!NOTE] US-001: Структура навигации (Main vs Settings)
> **Как** пользователь,
> **Я хочу** видеть разделение между основным функционалом (управление заданиями) и настройками профиля,
> **Чтобы** иметь быстрый доступ к обучению и не отвлекаться на второстепенные функции.
>
> **Критерии приемки (AC):**
> - [ ] **Меню Пользователя (Reply Keyboard):** Содержит кнопки доступа к Профилю и Настройкам. Всегда доступна "над клавиатурой".
> - [ ] **Основной Функционал (Inline Keyboards):** Выбор режимов (Спринт, Тренировка), выбор тем и само решение задач происходит через кнопки "под сообщением".
> - [ ] Главное меню вызывается командой `/start` и отправляет сообщение с Inline-кнопками для выбора режима обучения.

  
  

> [!TODO] US-002: Режим "Спринт"
> **Как** пользователь, у которого мало времени,
> **Я хочу** иметь возможность быстро прорешать фиксированный набор (10-20) несложных задач с мгновенным фидбеком,
> **Чтобы** поддерживать тонус знаний, занимаясь короткими сессиями "на ходу".
>
> **Критерии приемки (AC):**
> - [ ] Режим запускает сессию из 10-20 вопросов.
> - [ ] Уровень сложности задач: Лёгкий / Средний (без сложных вычислений).
> - [ ] После каждого ответа сразу показывается: Правильно/Неправильно.
> - [ ] Если ответ неверный — отображается верный ответ и краткое решение.
> - [ ] В конце спринта показывается итог (например, 8/10 верно).
> - [ ] **Отображение прогресса:** В каждом вопросе показывается "Вопрос N из Total" (например, 3/10).
> - [ ] **Выход:** При переходе в Главное меню или другие разделы текущая сессия (Спринт/Тренировка) сбрасывается без сохранения.

  

> [!TODO] US-003: Режим "Тренировка" (Адаптивный)
> **Как** пользователь, нацеленный на глубокое изучение,
> **Я хочу** решать бесконечный поток задач по выбранным темам с постепенным повышением сложности,
> **Чтобы** глубоко проработать материал и закрыть пробелы в знаниях.
>
> **Критерии приемки (AC):**
> - [ ] Возможность выбора конкретных тем перед стартом (чекбоксы).
> - [ ] Режим работает без ограничения по времени или количеству (Endless mode).
> - [ ] Алгоритм предлагает задачи по возрастанию сложности (от простых к сложным).
> - [ ] При ошибке отображается полное развернутое решение.
> - [ ] Возможность прервать тренировку в любой момент с сохранением прогресса.

  
  

> [!TODO] US-004: Каталог тем и теория
> **Как** пользователь,
> **Я хочу** иметь список всех тем с возможностью выбора их для тренировок, а также возможностью зайти внутри темы, прочитать теорию и прорешать все задачи списком (от простых к сложным),
> **Чтобы** гибко настраивать своё обучение: либо смешивать выбранные темы в спринтах и тренировка, либо методично изучать одну конкретную область.
>
> **Критерии приемки (AC):**
> - [ ] Экран "Темы" со списком всех доступных разделов.
> - [ ] Переключатели напротив тем для включения их в режим "Спринт" и "Тренировка".
> - [ ] При клике на саму тему открывается её карточка:
>     - [ ] Вкладка "Теория" (текст/формулы).
>     - [ ] Вкладка "Задачи" (список всех задач темы).
> - [ ] Сортировка задач внутри темы по возрастанию сложности.
> - [ ] Визуальная отметка прорешанных задач в списке.

  

> [!TODO] US-005: Режим "Экзамен"
> **Как** пользователь, проверяющий свои знания,
> **Я хочу** пройти сбалансированный тест по выбранной теме с заданной сложностью, где результаты и ошибки будут показаны только после завершения всего теста,
> **Чтобы** оценить свою реальную готовность и умение решать задачи без подсказок и мгновенной проверки.
>
> **Критерии приемки (AC):**
> - [ ] Выбор уровня сложности перед стартом (Лёгкий / Средний / Сложный / Хардкор).
> - [ ] Генерация сбалансированного варианта (разные типы задач внутри темы).
> - [ ] Отсутствие кнопки "Проверить" после каждого вопроса (только кнопка "Следующий вопрос").
> - [ ] Возможность вернуться к предыдущему вопросу до завершения экзамена.
> - [ ] Экран результатов (Score screen) появляется только после отправки всего экзамена.

  

> [!TODO] US-006: Работа над ошибками
> **Как** пользователь, стремящийся закрыть пробелы в знаниях,
> **Я хочу** видеть список своих ошибок с разбивкой по темам и иметь возможность перерешать эти задачи (как все подряд, так и по конкретной теме),
> **Чтобы** не повторять одни и те же ошибки в будущем и видеть статистику улучшения.
>
> **Критерии приемки (AC):**
> - [ ] Отдельный раздел "Мои ошибки".
> - [ ] Отображение общей статистики (например: "Всего ошибок: 15").
> - [ ] Фильтрация ошибок:
>     - [ ] "Все подряд" (микс из всех тем).
>     - [ ] "По конкретной теме" (выбор папки).
> - [ ] Режим перерешивания: пользователю предлагается задача, в которой он ошибся ранее.
> - [ ] При правильном решении задача исчезает из списка "Ошибки" (или помечается как исправленная).

>
> **Критерии приемки (AC):**
>
> - [ ] Команда или кнопка «Профиль» отображает карточку пользователя.
>    
> - [ ] Отображение текущего уровня (например, "Новичок", "Профессионал").
>    
> - [ ] Счетчик «Всего задач выполнено».
>    
> - [ ] Визуальная или текстовая шкала прогресса до следующего уровня (XP bar).
>    
> - [ ] Показатель «Точность решений» (процент правильных ответов).
>

# MVP Scope

| Фича (User Story) | MVP? | Почему? |
| :--- | :---: | :--- |
| **US-001: Навигация (Main vs Settings)** | ✅ | **Критично.** Пользователь должен сразу понимать, куда жать. Без разделения интерфейс будет перегружен. |
| **US-002: Режим "Спринт"** | ✅ | **Core Value.** Реализация "микрообучения" и коротких сессий, ради которых создается продукт. |
| **US-003: Режим "Тренировка" (Адаптивный)** | ✅ | Необходимо для более глубокого погружения в тему. |
| **US-004: Каталог тем и теория** | ✅ | **База.** Пользователь должен выбирать, *что* он учит. Можно упростить структуру на старте. |
| **US-005: Режим "Экзамен"** | ❌ | **Избыточно.** В v1 достаточно "Спринта". "Экзамен" — фича для удержания в v1.1. |
| **US-006: Работа над ошибками** | ✅ | **Необходимо.** Одна из ключевых фич для обучения и удержания. |
| **US-007: Профиль пользователя** | ✅ | **Retention.** Нужна минимальная статистика (XP/Счетчик) для чувства прогресса. |
| **US-008: Панель администратора (Меню)** | ✅ | **Ops.** Позволяет управлять ботом без привлечения разработчика. |
| **US-009: Управление темами/контентом** | ✅ | **Content is King.** Критически важна удобная (в т.ч. массовая) загрузка контента. |
| **US-010: Управление пользователями** | ❌ | **Рано.** На старте пользователей мало. Модерацию можно делать напрямую через БД. |
| **US-011: Система рассылки** | ✅ | **Маркетинг.** В мессенджерах жизненно важно возвращать пользователей пушами. |
